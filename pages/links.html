Software:

Much of the work I do is using Python for scientific computing. Besides the usual suspects (NumPy, SciPy, Matplotlib, and IPython), I also make use of:

  • SymPy — computer algebra system
  • PyDy — tools for dynamic systems analysis
  • Cython — easy, fast C extensions
  • Numba — speed up code using an LLVM based jit
  • Numexpr — speed up array expressions
  • Toolz — a better functional library

I've also been using Julia a lately. While it took me a bit to get used to the language, I find it to be very performant, while still remaining easy to write (and read!). Some Julia libraries I like:

  • Gadfly — plotting, similar to ggplot2
  • DataFrames — tabular data analysis
  • Convex — disciplined convex programming

As far as workstation, I run Arch Linux, and make heavy use of the shell. As such, I spent some time customizing my workflow. My dotfiles can be found here, although they're probably pretty poorly written. I use tmux as my terminal multiplexer, and tmuxinator to store common window setups. I switched to vim a few years back, and am addicted (so much so that I invariably type ":w" into word documents). Git is my version control tool of choice.
